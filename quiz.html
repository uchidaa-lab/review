Q1 🏠 研修の根本
「仕事とは何ですか？」を一言で答えてみよう。
また、TOKIUMが伝えた「目指す人物像」の4要素は？
こたえ
仕事とは → ステークホルダーを安心させること

目指す人物像の4要素 → 「5年後に」「組織を牽引する」「異常な情熱を持つ」「リーダー」
「5年後」と「組織を牽引」が抜けやすいので注意！
Q2 💪 6動詞
ビジネスの基本動作「6つの動詞」を全て挙げてみよう。
「学ぶ」の定義は何だった？
こたえ
6動詞 → 学ぶ・聴く・読む・書く・話す・巻き込む

「学ぶ」の定義 → 教わったことを、翌日、自力で100%再現できること
「100%再現」がキーワード！復習 vs 新しい本の問題では「バケツの穴」「アクティブリコール」にも触れよう
Q3 ☂️ 空・雨・傘
空・雨・傘の3層はそれぞれ何を表す？
「なぜ3層に分けて伝えるの？」の目的も答えてみよう。
こたえ
空 = 事実　雨 = 解釈　傘 = 提案

目的 → 上司の思考コストを下げて、信頼残高を貯めるため
傘（提案）には「何を」「いつまでに」を入れるのがポイント！
Q4 🔍 具体と抽象
具体的な行動を書くときに使ってはいけないNGワードは？
代わりにどんな形で書くべき？
こたえ
NGワード → 「意識する」「気をつける」など曖昧な表現

代わりに → 「誰が・いつまでに・何を」が明確で、すぐ動ける粒度で書く
例：「明日から会議のたびにノートを開いて決定事項を1行ずつ記録する」のように具体的に！
Q5 ✍️ 簡潔にロジカル
PREP法の4つの要素と、文章を簡潔にする4原則を挙げてみよう。
こたえ
PREP法 → P（結論）→ R（理由）→ E（具体例）→ P（再結論）

4原則 → ①1文1メッセージ ②接続詞を8割削る ③修飾語を削る ④能動態で書く
上司の時間を奪わないため＆後回しにされないために圧縮する！
Q6 ✅ アクション返答
上司に仕事を頼まれたとき、「了解しました」の返事に必要な3要素は？
こたえ
①何の件か ②承知しました ③何をいつまでにするか

例：「アンケート集計の件、承知しました。本日18時までに集計してお送りします。」
「承知しました」だけだと上司が「いつ届くの？」と追加確認する羽目に…
Q7 📐 20%ルール
骨子の3要素は？ 手戻りを防ぐための3ステップも答えてみよう。
こたえ
骨子の3要素 → タイトル・結論・構成

3ステップ → ①骨子を先に作る ②方向性確認を依頼する ③合意後に作り込む
100%作ってから見せるのはリスク大！20%の段階で方向性を確認しよう
Q8 🤔 仮説付き質問
仮説付き質問の3つの型と、それぞれの「口癖」を言えるかな？
こたえ
①検証型 →「私はAだと思うのですが、合っていますか？」
②比較提示型 →「BとCで迷っています。〇〇の理由でBが良いと思いますが…」
③根拠参照型 →「資料の△△に基づき、今回も適用可能と考えますが？」
相手がYes/Noで答えられる形にするのがコツ！
Q9 🙏 相手の立場を想像した依頼
誰かに何かをお願いするとき、依頼文に必要な4要素は？
こたえ
①専門性への敬意（なぜあなたに頼むのか）
②主導権の譲渡（日程の調整余地を持たせる）
③断る余地（「繁忙期が終わってからでも構いません」など）
④感謝の明文化（ちゃんとお礼を言葉にする）
得点率34%の苦手テーマ。相手の多忙さや立場に配慮できているかがカギ！
Q10 📣 報連相の技術
報連相の「第一声」で何をすべき？
理想の報連相メッセージに必要な3要素は？
こたえ
第一声 → 「報告/連絡/相談」のどれかを宣言する

3要素 → ①種別宣言 ②結論ファースト ③自分なりの仮説を添える

NG → 丸投げ相談（「どうしたらいいですか？」）
「ご相談です。〇〇と考えていますがいかがですか？」の形がベスト！
Q11 🚨 バッドニュース15分ルール
一次報告テンプレートの4要素は？
（みんなの得点率26%、一番むずかしかった問題！）
こたえ
①状況（今何が起きているか）
②原因（なぜ起きたか）
③経過（これまでの対応）
④対応案（これからどうするか）

冒頭で「バッドニュースの報告です」と宣言 ＋ 15分以内に報告！
「すぐ報告する」は書けても4要素が出てこない人がほとんどだった。ここは確実に覚えよう！
Q12 🎯 ネクストアクション
ネクストアクション（NA）の4要素を答えてみよう。
こたえ
①Who（誰が）
②When（いつまでに）
③What（何を）
④Done条件（何をもって完了とするか）
「次回までに進捗共有」→ NG！「Mさんが4/15 17:00までにデザイン案をSlackに投稿し、全員からOKを得る」→ OK！
Q13 🪞 振り返りの技術
振り返りに必要な4要素は？
また、絶対に使ってはいけないNGワードは？
こたえ
4要素 →
①問題点指摘（何がうまくいかなかったか）
②Why分析（なぜうまくいかなかったか）
③因数分解（量の問題か質の問題か）
④具体トライ（来週何をどうするか）

NGワード → 「頑張る」「しっかり」「気をつける」
NGワードを使うと原因が特定できず、同じ失敗を繰り返しちゃう
Q14 🚀 「新卒」を脱ぎ捨てる
「受け身はまずい」のはなぜ？
TOKIUMの組織特性に触れて説明してみよう。
こたえ
TOKIUMは「スピード100で走りながら整える」組織。不確実な環境で自分から動き出す力が求められるため、100%整うのを待つ受け身姿勢では配属初日に止まってしまう。
「他責はダメ」だけだと不十分。TOKIUMの組織特性（走りながら整える＋不確実な環境）に触れよう！
Q15 🔥 食らいつく精神
次のケース、「回避型」と「食らいつき型」どっち？
A. 会議中に黙って後でこっそり調べた
B. テストの間違いをその日のうちにメモなしで確認した
こたえ
A. → 回避型（その場で聞くべき！後で調べても会議中のズレは取り返せない）

B. → 食らいつき型（アクティブリコールの実践。間違いをその日のうちに潰してる！）
「次回がんばろう」も回避型。いつ・何をするか決まってないと食らいつけてない！
Q16 🤝 他者を動かす
「やり方に不満がある」とき、愚痴で終わらせず行動に変える3ステップは？
こたえ
①WHYを理解する（まず「なぜこうしているのか」を相手に聞く）
②空雨傘で構造化する（問題と改善案を整理して伝える）
③小さく始める（いきなり全部変えずに、まず1ヶ月データを取ってみる）
愚痴は行動を伴わない。動かなければ何も変わらない！
Q17 ⏩ 前/後ろの行動
次のケース、チームを「前に進める」行動？「後ろに引っ張る」行動？
A. データに基づいてリスクを指摘した
B. 他のタスクがあるのに引き受けて遅延した
C. 懸念があったけど黙っていてトラブルになった
こたえ
A. → 前に進める（データに基づく指摘はチームの判断の質を上げる！）

B. → 後ろに引っ張る（引き受ける前に優先順位を確認すべき）

C. → 後ろに引っ張る（懸念を共有しないと、チームの判断に必要な情報が欠ける）
判定が分かれるケースもある。「前」「後ろ」どちらでも理由が合理的であればOK！
